# Story Agent UI 设计系统与后续修改规范

状态：正式实施规范

视觉名称：**墨曜指挥舱**

适用范围：Story Agent 桌面 Web、未来 WebView2 桌面封装

目标窗口：1440×1024、1280×800

## 1. 设计定位

Story Agent 是高密度、长时间使用的专业故事生产工具，不是普通企业后台，也不是聊天机器人套壳。

视觉关键词：

- 墨蓝、克制、冷静；
- 暖金表示作者意图、关键操作和当前焦点；
- 靛蓝表示 AI/模型活动；
- 绿色表示通过或安全；
- 琥珀表示等待、风险或需要注意；
- 红色只表示阻断、失败或不可逆危险；
- 信息密度高，但层级、边界和下一步必须清楚。

任何后续页面都必须让用户一眼分辨：

1. 当前是哪部作品、哪一章、哪个作用域；
2. 当前处于草稿、候选、待确认还是正式状态；
3. 当前最重要的下一步操作是什么；
4. 哪些内容是人工确定，哪些来自 AI；
5. 为什么按钮不可用，以及去哪里修复。

## 2. 设计令牌

当前权威实现位于 `apps/web/src/styles.css`。新增 CSS 必须优先复用令牌，禁止在组件中随意创造相近颜色。

### 2.1 色彩

| Token | 数值 | 用途 |
|---|---|---|
| `--bg-deep` | `#06101c` | 应用最底层背景 |
| `--bg-app` | `#081421` | 中央工作区背景 |
| `--bg-panel` | `#0a1725` | 标准面板、卡片 |
| `--bg-raised` | `#0d1b2a` | 弹窗、悬浮、选中层 |
| `--bg-input` | `#081522` | 输入框、编辑器 |
| `--line` | `#233344` | 主要分隔线、面板边框 |
| `--line-soft` | `#192a3a` | 次级分隔线 |
| `--text` | `#d8dde4` | 正文和常规标签 |
| `--text-strong` | `#f1eee7` | 标题、关键值 |
| `--muted` | `#7e8b99` | 辅助说明 |
| `--faint` | `#536474` | 时间、ID、弱提示 |
| `--gold` | `#d4a45f` | 主操作、作者焦点、当前选中 |
| `--gold-soft` | `#9d7845` | 金色边框、悬停 |
| `--gold-bg` | `#2b251d` | 金色选中背景 |
| `--indigo` | `#6c7cff` | AI、模型流式运行和提案 |
| `--green` | `#50b77a` | 成功、通过、已提交 |
| `--green-bg` | `#10291e` | 成功背景 |
| `--amber` | `#e5a83d` | warning、等待、费用风险 |
| `--red` | `#ef6b67` | blocker、error、失败、危险操作 |

色彩规则：

- 金色不能大面积铺满，只用于主按钮、当前焦点、标题和少量边框。
- 红色不能用于装饰；只有真正阻断或危险才使用。
- 状态不能只靠颜色，必须同时有图标和中文文字。
- 不引入高饱和霓虹紫、荧光蓝、大面积纯黑或白色后台风格。
- 渐变只能轻微用于选中、进度或氛围背景，不做炫光和玻璃拟态堆叠。

### 2.2 字体

| 场景 | 字体 |
|---|---|
| 中文正文、表单、按钮 | `Noto Sans SC Variable`, `Microsoft YaHei`, sans-serif |
| 页面标题、书名、章节标题 | `Noto Serif SC`, serif |
| JSON、revision、checksum、错误码 | `ui-monospace`, `Consolas`, monospace |

建议字号：

- 顶部作品名：20—24px；
- 页面主标题：28—30px；
- 分区标题：16—18px；
- 正文/按钮：12—14px；
- 辅助说明：11—12px；
- 不新增低于 10px 的用户可读文字。

旧页面存在为高密度原型保留的 7—9px 辅助文字；后续修复应提高可读性，不能继续扩散更小字号。

### 2.3 圆角、边框和阴影

- 全局圆角：`--radius: 6px`；
- 小控件：3—4px；
- 卡片/弹窗：5—8px；
- 禁止大面积 18—28px 的消费级“大圆角卡片”；
- 面板主要依靠 1px 边框分层，阴影只用于弹窗、浮层和重要悬浮对象；
- 不给每张卡片都加重阴影。

### 2.4 间距

以 4px 为基础单位：

- 紧凑：4 / 6 / 8px；
- 标准组件内边距：10 / 12 / 16px；
- 页面区块：20 / 24px；
- 页面标题与内容：16—20px；
- 同一层级间距保持一致，不通过随机空白制造层级。

## 3. 应用外壳

桌面端固定三栏：

```text
左侧主导航 | 中央业务工作区 | 右侧固定故事 Agent
              顶部项目状态栏
              页面内容
              底部全局状态栏
```

权威尺寸：

- 左侧导航：`112px`；1380 以下收窄为 `94px`；1180 以下为 `86px`；
- 顶栏：`60px`；
- 底栏：`62px`；
- 右侧 Agent 默认：`374px`；
- 应用最小宽度：`1024px`；
- 中央内容始终使用 `minmax(0, 1fr)`，防止内部内容撑破全局布局。

1180px 以下：

- 右侧 Agent 优先折叠为 48px；
- 中央业务区不能被 Agent 遮挡；
- 时间轴、章节轨道等确需宽度的组件可在自身容器滚动；
- 禁止让整个页面产生不可控横向滚动。

## 4. 页面层级

每个正式页面统一包含：

1. 英文小型 kicker，用于模块气质，不承担核心说明；
2. 中文页面标题；
3. 一句“本页解决什么问题”的中文说明；
4. 当前状态/就绪状态；
5. 主要工作区；
6. 明确的主操作；
7. 失败、空数据、加载和恢复状态。

页面不得只放一组无意义统计卡片。数据卡只在它能帮助当前决策时出现。

## 5. 导航与模块身份

| 模块 | 主要视觉/交互重点 |
|---|---|
| 作品总览 | 书架、作品状态、下一步引导；创建是主操作 |
| 创意孵化 | 六阶段轨道、证据与作者决定分离、三开篇对比 |
| Canon | 锁定感、实体/规则/关系、正式与变更申请明确分离 |
| 故事规划 | 时间轴、里程碑、范围和依赖；金色表示当前预算焦点 |
| 章节写作 | 章节轨道、契约、正文、版本和流水线三列协作 |
| 质量中心 | 严重度、证据、修复和提交门；红色只用于真实阻断 |
| 自动托管 | 1/3/5 章入口、队列、费用、恢复和就绪检查 |
| 模型设置 | Provider → 模型 → 角色绑定的清晰顺序 |
| 安全审计 | 备份、事件、调用、诊断；强调可追溯性 |

`/state` 和 `/drama` 仍是占位模块。未正式开发前不得伪装成可用页面。

## 6. 组件规范

### 6.1 按钮

按钮层级：

1. **主操作**：暖金边框/背景，例如“生成”“应用”“质量通过并提交”；每个局部区域最多一个。
2. **普通操作**：墨蓝背景、蓝灰边框，例如“保存草稿”“测试连接”。
3. **文本操作**：透明背景，例如“查看详情”“撤销”。
4. **危险操作**：红色边框/文字，例如“删除”“停止”“拒绝”；必要时二次确认。

规则：

- disabled 必须有就近原因，不能只变灰。
- 异步按钮点击后立即显示 loading 文案/动画，并阻止重复提交。
- 按钮文案使用“动作 + 对象”：`锁定 Canon`、`生成章节契约`，不用模糊的“确定”。
- 图标是辅助，不能代替中文文字；纯图标按钮必须有 `aria-label` 和 tooltip。

### 6.2 表单

- 标签始终可见，不用 placeholder 代替字段名。
- 重要字段说明写在字段旁或下方，不放到远处的帮助文档。
- JSON 编辑器使用等宽字体，并显示示例或校验错误。
- 保存前做前端基本约束；服务端错误必须显示中文信息和 request ID/错误码。
- 有未保存修改时必须显示 dirty 状态，切换页面前应提示。

### 6.3 卡片和面板

- 标准面板使用 `--bg-panel` + `--line`；
- 选中面板使用金色边框/顶部细线，不大面积填充金色；
- 卡片头部左侧为标题/图标，右侧为状态或操作；
- 空状态应说明“为什么为空”和“下一步做什么”；
- 错误状态应说明可恢复动作，不能只输出英文错误码。

### 6.4 表格、列表和时间轴

- 当前行、选中行和悬停行必须可区分；
- 表头保持稳定，长内容使用省略或内部滚动；
- 状态列同时显示颜色点/图标与文字；
- 时间轴用范围和位置表达节奏，不用装饰性折线图替代真实章节坐标；
- 章节与里程碑必须保留范围、目标、依赖和状态语义。

### 6.5 对话框、Toast 和确认

- 对话框只用于需要集中确认的操作；
- Canon 锁定、正式提交、删除、恢复等高影响操作必须明确影响范围；
- Toast 只确认短暂结果，不承载长错误详情；
- 失败信息保留在页面中，刷新后从业务状态恢复；
- 不使用浏览器原生 alert 代替正式业务错误页面（简单二次确认除外）。

## 7. 状态语义

| 状态 | 颜色 | 必备文字 |
|---|---|---|
| ready / passed / completed | 绿色 | 已就绪、通过、已提交 |
| running / selected / AI active | 靛蓝或金色 | 运行中、当前、生成中 |
| queued / warning / awaiting review | 琥珀 | 排队、待复核、风险 |
| blocked / failed / error | 红色 | 阻断、失败、错误原因 |
| draft / inactive / not planned | 蓝灰 | 草稿、未启用、未规划 |

`superseded` 必须明确显示“旧版本问题已被替代”，不能继续以活跃红色误导用户。

## 8. 人工与 AI 的视觉边界

- 普通聊天不得表现成已经修改正式数据。
- AI 输出先进入候选稿或 `ChangeProposal`。
- 差异必须显示 before/after、原因和下游影响。
- 用户可以逐项接受、全部接受、拒绝和撤销。
- 正式 Canon、规划和章节提交必须显示锁定/版本/revision。
- AI 生成状态使用靛蓝；作者确认和正式焦点使用暖金；正式通过使用绿色。
- 不得用“AI 已完成”掩盖真实模型失败，也不得静默回退固定模板。

## 9. 固定右侧故事 Agent

右侧 Agent 是应用外壳的一部分，不是某一页的临时聊天框。

必须保留：

- 当前作品、页面、章节、契约、实体或选区作用域；
- 页面相关快捷动作；
- 流式运行状态、停止、失败重试；
- 结构化提案差异和影响；
- 接受、拒绝和撤销；
- 折叠/展开与宽度偏好；
- 页面切换后业务对话从后端恢复。

响应式优先级：当宽度不足，先折叠 Agent，不能压缩正文到不可操作或遮住按钮。

## 10. 可访问性和可理解性

- 重要状态不只依赖颜色；
- 键盘焦点使用 `2px #89a3ff` 明确轮廓；
- 点击目标建议至少 32px 高，主要按钮 34—40px；
- tooltip 只能补充，不应隐藏核心说明；
- 专业术语首次出现必须给中文解释，例如 `Canon（正式设定库）`；
- 范围按钮应解释“第 6—6 章 = 只写第 6 章”；
- 禁用按钮旁必须显示阻断检查和修复入口；
- 任何空页面都应告诉用户推荐入口，不能只有“敬请期待”。

## 11. 动效

- 过渡时间通常 120—200ms；
- loading 使用旋转或轻微脉冲；
- 不使用大面积弹跳、粒子、炫光或长时间背景动画；
- 尊重 `prefers-reduced-motion`；
- 动效不能延迟业务操作反馈。

## 12. UI 修改工作流

修改前：

1. 读取本规范、`UI-DESIGN-BASELINE.md` 和 `design-qa.md`；
2. 打开 [图文操作手册](../Story-Agent-使用手册.html) 确认用户流程；
3. 截取改动页面 1440×1024 和 1280×800 当前基线；
4. 确认修改范围，不顺手重构其他页面。

实现时：

- 复用全局 token、现有 CSS 类和 Phosphor 图标；
- 保留三栏外壳、顶部状态栏、底部状态栏和右侧 Agent；
- 业务状态来自 API/React Query，不把正式内容写入 `localStorage`；
- 新增状态必须覆盖 loading/empty/error/disabled/success/409；
- 不修改视觉快照以“让失败消失”，先判断实际 UI 是否退化。

修改后：

```powershell
npm run build
npm run test:web
npm run test:e2e
git diff --check
```

并人工检查：

- 1440×1024；
- 1280×800；
- 125% Windows 缩放（条件允许时）；
- 中文是否完整、无乱码；
- 按钮是否可理解；
- 弹窗是否超出窗口；
- 长标题、长错误、长正文和长 JSON 是否安全换行/滚动；
- Agent 折叠后中央区域仍能操作。

## 13. 禁止事项

- 禁止改成白底、浅灰卡片、蓝色主按钮的通用后台。
- 禁止整体更换主色、字体和图标体系。
- 禁止取消固定右侧 Agent 或让它遮住工作区。
- 禁止把人工编辑入口全部移入聊天框。
- 禁止 AI 未确认就直接更新正式 Canon、规划或正文。
- 禁止只显示英文错误码或把失败伪装成成功。
- 禁止堆砌无意义统计卡片、超大圆角、重阴影和过度玻璃拟态。
- 禁止为了塞更多内容继续缩小字号。
- 禁止 UI 修改顺带改数据库事务、迁移历史或模型安全边界。

## 14. 视觉验收清单

- [ ] 仍然一眼可识别为“墨曜指挥舱”，没有退化为普通后台。
- [ ] 暖金只用于主操作和作者焦点。
- [ ] AI、通过、警告、失败四类状态语义一致。
- [ ] 当前作品、章节和作用域始终可见。
- [ ] 主操作明确，禁用原因就近可见。
- [ ] 所有 AI 修改先显示候选或差异。
- [ ] 1440×1024 无遮挡、无全局横向溢出。
- [ ] 1280×800 Agent 自动折叠或不遮挡中央操作。
- [ ] 加载、空数据、失败、恢复、409 和刷新恢复均可理解。
- [ ] 现有未修改页面视觉快照不发生意外变化。
- [ ] 用户手册涉及的按钮名称与实际界面同步。
